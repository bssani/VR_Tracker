# VTC 캘리브레이션 정확도 설명 자료
> Leadership 발표용 — 캘리브레이션 원리, Hip Position 적용 방식, 정확도 근거, 경쟁 모델 비교

---

## 1. 캘리브레이션 전체 흐름

### 트리거
운영자가 키보드 `1`을 누르면 캘리브레이션 시작.

### 단계별 설명

```
[키 1 입력]
    ↓
SessionManager: Idle → Calibrating 상태 전환
    ↓
모든 트래커 활성화 여부 확인 (5/5 연결 필수)
    ↓
피험자에게 T-Pose 지시 (양팔 수평, 발 어깨너비)
    ↓
3초 카운트다운 (음성 피드백: "3", "2", "1", "완료!")
    ↓
SnapCalibrate() 실행 — 5개 트래커 위치 동시 스냅샷
    ↓
CalculateMeasurements() — 4개 신체 세그먼트 거리 계산
    ↓
ValidateMeasurements() — 해부학적 타당성 자동 검증
    ↓
검증 통과 → bIsCalibrated = true, 테스트 활성화
검증 실패 → 실패 원인 출력, 재시도 요청
```

### 측정되는 4개 세그먼트

| 세그먼트 | 설명 |
|----------|------|
| Hip → LeftKnee | 좌측 대퇴골 길이 (Waist 트래커 → 좌측 무릎 트래커) |
| Hip → RightKnee | 우측 대퇴골 길이 |
| LeftKnee → LeftAnkle | 좌측 경골 길이 (좌측 무릎 → 좌측 발목 트래커) |
| RightKnee → RightAnkle | 우측 경골 길이 |

추가로 HMD 높이로 피험자 신장을 추정 (`EstimatedHeight = HMDHeight / 0.92`).

## 2. Hip Position 적용 방식

### VehicleHipPosition이란?

차량 설계 기준으로 정해진 **운전자 Hip의 목표 위치** (월드 좌표, cm).
INI 파일(`Config/VTCSettings.ini`)에 `VehicleHipPosition_X/Y/Z`로 저장되며,
세션 시작 시 자동 로드 → BodyActor에 적용된다.

### bMonitorOnly = true 의미

`VehicleHipPosition`은 Warning/Collision을 발생시키지 않는다.
순수하게 **Waist 트래커 ↔ VehicleHipPosition 간 거리를 실시간 측정·기록**하는 용도로만 사용된다.
이를 통해 피험자가 설계 기준 Hip 위치에서 얼마나 벗어나 있는지 정량화한다.

### Hip Snap (키 4) 동작 원리

```
[키 4 입력]
    ↓
현재 Waist 트래커 위치 읽기 (MountOffset 적용 후)
    ↓
Delta = VehicleHipPosition - CurrentWaistEffectiveLocation
    ↓
Pawn 전체를 Delta만큼 이동
    ↓
결과: 피험자 Hip이 차량 설계 기준점에 정확히 정렬됨
```

**목적**: 모든 피험자를 동일한 기준 Hip 위치에서 시작시켜
개인 체형이 아닌 **차량 설계 기준 대비 간섭 여부**를 비교 가능하게 한다.

## 3. MountOffset 보정 — 정확도의 핵심

Vive Tracker는 신체에 직접 부착되지 않고 스트랩/클립을 통해 고정된다.
트래커 물리적 중심 ≠ 신체 실제 접촉점이므로, 오프셋 보정이 필수.

### 보정 수식

```
EffectiveLocation = TrackerWorldLocation
                  + Quaternion(TrackerWorldRotation) × MountOffset
```

트래커가 어떤 방향을 향하든, **트래커 로컬 좌표의 오프셋이 올바른 월드 위치로 자동 변환**된다.
설정값은 Details 패널 또는 INI 파일의 `MountOffset_{Part}_X/Y/Z`로 입력.

## 4. 거리 계산 알고리즘 (CollisionDetector)

30Hz 주기로 실행. 단순 트래커 위치 간 거리가 아닌,
**신체 볼륨을 반영한 Surface-to-Surface 거리**를 사용:

```
SurfaceDistance = max(0, CenterDistance − BodyPartRadius)
```

| 신체 부위 | Sphere 반경 (기본값) | 근거 |
|----------|---------------------|------|
| Waist (Hip) | 12 cm | 골반 외곽선 |
| Knee | 8 cm | 무릎 외측 돌출부 |
| Ankle | 10 cm | 발목 외측 복숭아뼈 |

단순 점 거리가 아닌 **실제 신체가 차지하는 공간 기준**으로 간격을 계산하므로
"신체가 물리적으로 차량에 닿기까지 남은 여유 공간"을 직접 나타낸다.

## 5. 정확도 근거 (Leadership Q&A 대비)

| 정확도 요소 | 구체적 내용 |
|------------|------------|
| **T-Pose 기반 측정** | 알려진 해부학적 기준 자세에서 측정 → 임의 자세 대비 오차 최소화 |
| **다중 유효성 검증** | 세그먼트 최소 10 cm / 좌우 비대칭 25% 미만 / 대퇴:경골 비율 0.7–1.5 범위 — 3중 검증 통과 시에만 캘리브레이션 확정 |
| **MountOffset 보정** | 트래커 마운트 편차를 수학적으로 제거, 트래커 회전 변화에도 항상 올바른 신체 위치 계산 |
| **Surface-to-surface 거리** | 신체 볼륨(Sphere 반경) 반영 → "신체와 차량 표면 사이의 실제 공간" 측정 |
| **30Hz 실시간 샘플링** | 동작 전 구간(탑승·착석·이탈) 중 최소 간격 포착 — 정적 측정 불가능한 순간을 기록 |
| **재현 가능한 CSV 기록** | 세션별 세그먼트 길이, 최소 클리어런스, 위치 좌표 전체 저장 → 데이터 검토·비교·재분석 가능 |
| **Hip Snap 표준화** | 모든 피험자가 동일 차량 기준 Hip 위치에서 시작 → 피험자 간 비교 유효성 확보 |

## 6. RAMSIS 등 정적 인체공학 모델과 비교

> RAMSIS (Realistic Anthropological Mathematical System) : BMW·VW 등 자동차 OEM이 좌석·조작계 설계에 사용하는 디지털 인체 모델.

### 비교 테이블

| 항목 | **VTC (이 시스템)** | **RAMSIS / 정적 인체 모델** |
|------|-------------------|---------------------------|
| **측정 방식** | 동적 — 실제 탑승·착석·하차 전 동작 궤적 추적 | 정적 — 특정 자세(JD38, SAE H-Point)의 퍼센타일 마네킹 스냅샷 |
| **개인화** | 피험자별 T-Pose 캘리브레이션 (실제 체형 측정) | 5th/50th/95th 퍼센타일 평균값, 개인 편차 미반영 |
| **측정 대상** | 동작 중 발생하는 최소 간격 (cm 실측) | 설계 자세에서의 예상 여유 공간 (시뮬레이션) |
| **동작 궤적** | 3D 경로 전체 기록 (탑승 동작의 knee swing 등) | 특정 착석 자세만, 동작 과정 없음 |
| **결과 형태** | 실측 cm값 + CSV 데이터 (ground truth) | 모델 예측값 (실제 사람 미검증) |
| **소요 시간** | VR 셋업 후 피험자 1명 기준 약 10분 | 3D 모델 구성 + 시뮬레이션 수 시간~수일 |
| **비용** | VR 장비(HTC Vive Pro 2 + 트래커 5개) 초기 투자 후 반복 측정 저비용 | 라이선스 비용 높음, 전문 엔지니어 필요 |

### 핵심 포지셔닝

```
RAMSIS (설계 단계)    →    VTC (검증 단계)
      ↓                         ↓
 "설계상 문제 없어야"         "실제로도 문제 없음을 실측으로 증명"
```

- **RAMSIS는 설계 초기 스크리닝에 최적**: 아직 차량이 없을 때 퍼센타일 마네킹으로 레이아웃 검증.
- **VTC는 검증 단계에서 강점**: 실제 사람이 실제 차량(또는 목업)에 탑승하는 동작에서 발생하는 최소 간격을 실측.
- **두 방법은 경쟁 관계가 아니라 상호 보완**: RAMSIS로 설계 → VTC로 실측 검증.

### VTC의 한계 (솔직한 언급 권장)

| 한계 | 내용 |
|------|------|
| VR 장비 필요 | HMD 착용 + 트래커 5개 부착 — 준비 시간 약 5분 |
| 트래커 부착 오버헤드 | 정확한 MountOffset 측정 필요 |
| 신체 볼륨 단순화 | Sphere 근사 — 실제 무릎·발목 형상과 차이 존재 |
| SteamVR 의존성 | 트래킹 정확도는 SteamVR 환경(base station 배치 등)에 영향을 받음 |

## 7. 발표 시 강조 포인트 요약

1. **"실제 사람, 실제 동작, 실측 cm값"** — RAMSIS가 줄 수 없는 것
2. **3중 유효성 검증**으로 잘못된 캘리브레이션 자동 차단
3. **Hip Snap**으로 모든 피험자를 동일 기준점에서 비교 — 통계적 유효성
4. **CSV 데이터**로 결과 재현 가능 — engineering 의사결정 지원
5. RAMSIS와 **대체 관계가 아닌 검증 단계 보완** 관계 — 기존 프로세스에 통합 가능
